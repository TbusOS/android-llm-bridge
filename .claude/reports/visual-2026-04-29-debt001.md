# DEBT-001 行为验证报告 · 2026-04-29

## 验证目标

F.6 ship 后，确认 LiveSessionCard.tpsSpark 从空 [] → 滚动窗口数据，
真正解决 DEBT-001（"LiveSession tps 退化为整段平均，spark 空"）。

## 验证手段

1. 起新版 alb-api（含 F.6 commit `606b88d`），点 ollama@10.0.25.46:11434
   gemma4:e4b
2. WS /chat/ws 跑一个真实 chat（用户问 adb / logcat 关系）
3. GET /audit?session_id=...&include_metrics=true 拉所有事件
4. Node 跑等同 useLiveSession reducer 逻辑（reducer 是纯函数，
   Node 跑结果 = React useMemo 跑结果），看 spark 数据

## 端到端发现的 P0 bug（已修）

`src/alb/api/audit_route.py:62 _project()` 把事件的 `data` 字段
silently dropped。导致：
- tps_sample 的 rate_per_s / total_tokens 拿不到
- tool_call_start 的 id / name 拿不到
- done 的 usage / model 拿不到

events.jsonl 里 data 是齐的，bug 在出口投影层。从 C.1（commit
36537d5，audit 升级为事件总线）就存在；C.5 ship LiveSession 框架
时没人真跑 chat 触发 tool_call，bug 静默到 F.6 端到端验证才暴露。

修法（一行）：`_project()` 加 `"data": raw.get("data")` 字段 +
TS 类型同步加 `data?: Record<string, unknown> | null`。

reducer 已正确：
- ✓ prompt 真实显示（来自 user 事件 + data）
- ✓ modelName 正确
- ✓ totalTokens 来自 sampler 累计（不是 done 整段平均）
- ✓ tps 是瞬时值（不是 done 时一次性算出的平均）
- ✓ tpsSamples 60s 滚动窗口
- ✓ tpsSpark scale 正确（peak normalize + clamp 0..36）
